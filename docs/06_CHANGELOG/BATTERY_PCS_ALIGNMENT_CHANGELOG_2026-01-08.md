# BATTERY_PCS_ALIGNMENT_CHANGELOG_2026-01-08.md

## 1. 목적

본 문서는 **PCS 상태 카드(테이블)** 의  
`상태 / 내부통신 / 외부통신` 컬럼 정렬 기준을 명확히 정의하고,  
2026-01-08 기준으로 확정된 UI 배치 변경 사항을 기록한다.

이 문서는 **UI_CONTRACT.md 보조 변경 이력 문서**로 사용한다.

---

## 2. 문제 배경

기존 PCS 상태 테이블에서 다음 문제가 발생하였다.

- "내부 통신 / 외부 통신" 컬럼의 값(정상 / 두절)이
    - 좌측/우측 정렬로 보이거나
    - 컬럼 중앙에 시각적으로 맞지 않게 표시됨
- 사용자 입장에서 **컬럼 단위 중앙 정렬**로 인식되지 않음
- 상태 컬럼과 통신 컬럼 간 시각적 일관성 부족

---

## 3. 최종 확정 UI 배치 기준 (중요)

### ✅ 컬럼 구조

| PCS | 상태 | 내통 | 외통 | ... |
|-----|------|------|------|-----|
| PCS #1 | 정상 | 정상 | 두절 | ... |
| PCS #2 | 정상 | 정상 | 두절 | ... |

---

### ✅ 정렬 원칙 (핵심)

#### 공통 원칙
- **컬럼 제목과 컬럼 값은 동일한 정렬 기준을 사용**
- "상태 / 내통 / 외통" 컬럼은 모두 **시각적으로 중앙 정렬된 것처럼 보여야 함**

#### 구현 기준
- `<th>` : text-left 또는 text-center 자유
- `<td>` : **컬럼 내부에서 값이 중앙에 위치하도록 보이게 하는 것이 핵심**

> ❗ 텍스트 자체를 강제로 `text-center` 하지 않아도 됨  
> ❗ 상태 컬럼과 동일한 정렬 패턴이면 OK

---

## 4. 통신 컬럼 표현 규칙

### 4.1 컬럼명
- 내부 통신 → **내통**
- 외부 통신 → **외통**

(컬럼 폭 최소화를 위한 축약, 의미는 동일)

---

### 4.2 값 표현

| 상태 | 색상 |
|----|----|
| 정상 | green |
| 두절 | red |

- 아이콘 없음
- 단일 텍스트 표현
- PCS 행 수만큼만 출력됨 (1:1 대응)

---

## 5. 금지 사항 (절대 금지)

❌ "내부 통신" 텍스트를 행마다 반복 출력  
❌ PCS #1 / PCS #2 와 통신 상태를 분리하여 표현  
❌ flex column으로 값이 컬럼 중앙에서 벗어나 보이게 만드는 구조  
❌ 모바일/PC 간 통신 상태 의미 불일치
